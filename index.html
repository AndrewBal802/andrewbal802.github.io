Problem
Take a open source project that is available for development and enhance the project. The open-source project chosen was the T-Rex Runner game which is a 2D endless runner game where a player controls a dinosaur to jump over or duck under obstacles for as long as they can. The visuals involved in the game are quite basic, it lacks creativity and may not be as appealing to most. Furthermore, the game is written in only one large Javascript file, resulting in poor modularization and poor software maintainability.
Solution
Developed the T-Rex runner game with Python using the PyGame Library. Several features of the game were added such as more obstacles, powerups and improved graphics. One major improvement to the originally game was redeveloping the game as a MVC design which increased the code quality and maintain ability. There was also a large focus on testing this system given the nature of an infinite runner game, most of the testing was done manually. However, some specifications from the Software Requirement Specification (SRS) such as performance was done through automated testing.
